1. How does Javascript work?

Answer: JavaScript is a single-threaded and single concurrent programming language, which means it has a single Call Stack and it can do one thing at a time. It's an interpreted programming language where an interpreter reads over the JavaScript code, interprets each line, and runs it. Also, like most other scripting languages, it uses dynamic typing, where type safety is verified at the runtime. For Asynchronous operations, promises are put into an event queue, which runs after the main thread has finished its processing so that they do not block subsequent JavaScript code from running. The queued operations will complete their tasks as soon as possible and then return the results to the JavaScript environment.

2. What does javascript event loop do?

Answer: Although JavaScript itself is single-threaded, the browser APIs act as separate threads. The event loop smoothens this process by executing the code, collecting and processing events, and executing queued sub-tasks. It constantly checks whether the call stack is empty or not. If it's empty, new functions are added from the event queue. If it's not, then the current function in call is processed.

3. What are the differences between Local Storage and Session Storage?

Answer: Local Storage and Session Storage are very similar to each other and have the same API. The only difference is that while data in Local Storage persists even when the browser/tab closes, in sessionStorage data is cleared when the page session ends. Data in local storage expires when the user manually clears the browser cache or when the web app clears the data. Also, when a document is loaded in a particular tab in the browser, a unique page session gets created and assigned to that particular tab. However, when an application needs data to be shared across multiple browser windows and tabs, Local Storage is used, otherwise, we can use Session Storage.
